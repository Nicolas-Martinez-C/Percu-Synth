# sample_loader — Cargador de samples (one-shots)

App web standalone que toma archivos de audio del navegador (drag & drop o file picker), los procesa (mono · 44.1 kHz · 16-bit · máximo 2 s) y **genera un firmware `.ino` con los samples embebidos en flash como arrays `PROGMEM`**. El sketch generado convierte al PercuSynth en un sampler polifónico de 5 pads disparables por los botones, con pitch shift controlado por potenciómetro.

> **Por qué generar un .ino y no cargar samples en runtime:** el ESP32-S3 no tiene almacenamiento masivo accesible en tiempo real desde un audio loop sin latencia. Embeber los samples en compile-time los pone en flash → acceso instantáneo desde el código.

## Cómo correrla

Abrir `index.html` directamente con doble-click funciona — esta tool no necesita Web Serial ni servidor. Recomendado usar **Chrome o Edge** porque son los que mejor soportan la Web Audio API que usa para previsualizar.

```bash
# Opcional, si querés servirla por HTTP:
cd tools/sample_loader
python -m http.server 8000
```

## Cómo se comunica con la placa

**No se comunica.** El flujo es:

```
audio files → webapp procesa → .ino generado → Arduino IDE → flash → placa
```

Una vez flasheado el firmware, la webapp no se vuelve a usar hasta querer cambiar los samples.

## Mapa del firmware generado

| Control          | Acción                                  |
|------------------|-----------------------------------------|
| BTN1 (GPIO 44)   | Dispara pad 1                           |
| BTN2 (GPIO 42)   | Dispara pad 2                           |
| BTN3 (GPIO 0)    | Dispara pad 3                           |
| BTN4 (GPIO 45)   | Dispara pad 4                           |
| BTN5 (GPIO 47)   | Dispara pad 5                           |
| POT1 (ADC1)      | Pitch (escala A frigia: A, Bb, C, D, E, F, G) |
| POT2 (ADC2)      | Filtro                                  |
| POT3 (ADC8)      | Resonancia                              |
| POT4 (ADC10)     | PWM (mod extra)                         |

**Mantener un BTN + mover POT1** cuantiza el sample a una nota de la escala frigia. Soltar y volver a apretar lo reproduce a la nota seleccionada.

## Flujo típico

1. Abrir `index.html` en Chrome.
2. Arrastrar hasta 5 archivos de audio sobre los pads (formatos: WAV/MP3/OGG/FLAC).
3. Click en cada pad → preview en el navegador antes de comprometer al firmware.
4. **GENERATE FIRMWARE (.ino)** → descarga `percusynth_samples.ino`.
5. Abrir en Arduino IDE → ESP32S3 Dev Module → USB CDC On Boot: Enabled → Flash Mode: DIO → upload.

## Limitaciones

- **Duración máxima por sample: 2 s** (sino no entran en flash con los 5 pads).
- Total recomendado: < 3.5 MB para que entre con margen junto al programa.
- Mono · 44.1 kHz · 16-bit (el procesamiento es automático).

## Output

El `.ino` generado se ve así (acortado):

```cpp

// Generated by PercuSynth Sample Loader

...
```
